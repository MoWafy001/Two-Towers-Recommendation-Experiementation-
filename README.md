# Multi-Modal Two-Tower Recommendation System PoC

A proof of concept for a recommendation system handling text, images, and audio using a Two-Tower architecture and local pretrained models.

## Core Concepts

### Embeddings
Content is converted into 512-dimension vectors (embeddings). Similar items have vectors that are mathematically close.

### Two-Tower Architecture
- **Item Tower**: Pretrained models (CLIP for text/image, CLAP for audio) generate embeddings for content.
- **User Tower**: User profile is represented by the average embedding of all items they have liked.

### Retrieval
Recommendations are generated by finding items whose embeddings are closest to the user's embedding using cosine similarity.

## Quick Start

1. **Start Services**
   ```bash
   docker compose up --build
   ```
   *Note: First run downloads ~1.2GB of models into the `models/` directory.*

2. **Access UI**
   [http://localhost:5173](http://localhost:5173)

## Project Structure

- `/backend`: Node.js API, interaction logic, and user tower calculation.
- `/frontend`: React UI for browsing and uploading.
- `/inference`: Python FastAPI service for model inference.
- `/models`: Local storage for CLIP and CLAP models.
- `/data/postgres`: Persistent database storage.
- `/data/uploads`: Storage for uploaded media.

## Examples

### Recommendation Query (SQL)
```sql
-- Find top 10 items closest to a user embedding
SELECT *, (embedding <=> $1) as distance 
FROM items 
ORDER BY distance + (random() * 0.2) 
LIMIT 10;
```

### User Embedding Update (Logic)
```javascript
// Average of liked item embeddings
const avgEmbedding = embeddings[0].map((_, i) => 
  embeddings.reduce((acc, e) => acc + e[i], 0) / embeddings.length
);
```
